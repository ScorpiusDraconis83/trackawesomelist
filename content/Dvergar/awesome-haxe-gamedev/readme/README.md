# Awesome Haxe Game Development [![Awesome](https://awesome.re/badge-flat2.svg)](https://awesome.re)

A curated list of game development resources for **[Haxe 4](https://haxe.org/)**, a high level strictly typed programming language which is used to produce cross-platform native code.

Feel free to update it.

## Game engines

Those are Haxe 4 compatible game engines

*   [Armory (Kha) (‚≠ê2.9k)](https://github.com/armory3d/armory) - An open-source 3D game engine with full Blender integration (`Web`, `Mobile`, `Desktop`, `Consoles`).
*   [Away3D (‚≠ê187)](https://github.com/openfl/away3d) - An open source, real-time 3D engine for OpenFL (`Web`, `Mobile`, `Desktop`).
*   [ceramic (‚≠ê211)](https://github.com/ceramic-engine/ceramic) - Cross-platform 2D framework (`Web`, `Mobile`, `Desktop`, `Unity`).
*   [HaxeFlixel (OpenFL) (‚≠ê1.8k)](https://github.com/HaxeFlixel/flixel) - Free, cross-platform 2D game engine powered by OpenFL (`Web`, `Mobile`, `Desktop`, `Consoles`).
*   [Haxegon (OpenFL) (‚≠ê192)](https://github.com/haxegon/haxegon) - A programming library for beginners. Powered by OpenFL and Starling (`Web`, `Mobile`, `Desktop`, `Consoles`).
*   [Heaps (‚≠ê3k)](https://github.com/HeapsIO/heaps) - High Performance Game Framework (`Web`, `Mobile`, `Desktop`, `Consoles`).
*   [hxdefold (‚≠ê177)](https://github.com/hxdefold/hxdefold) - Haxe/Lua externs for Defold game engine (`Web`, `Mobile`, `Desktop`).
*   [OpenFL (‚≠ê1.8k)](https://github.com/openfl/openfl) - Interactive game and app development library (`Web`, `Mobile`, `Desktop`, `Consoles`).
*   [Starling (‚≠ê230)](https://github.com/openfl/starling) - The "Cross-Platform Game Engine", a popular Stage3D framework (`Web`, `Mobile`, `Desktop`).
*   [Stencyl (OpenFL) (‚≠ê181)](https://github.com/Stencyl/stencyl-engine) - Create Flash, HTML5, iOS, Android, and desktop games with no code (`Mobile`, `Desktop`).
*   [unreal.hx (‚≠ê412)](https://github.com/proletariatgames/unreal.hx) - Haxe Integration for Unreal (`Web`, `Mobile`, `Desktop`, `Consoles`).
*   [HxGodot (Godot 4.0) (‚≠ê185)](https://github.com/HxGodot/hxgodot) - A Haxe GDExtension for Godot 4 (`Web`, `Mobile`, `Desktop`, `Consoles`).

## Low-level Engine

*   [Kha (‚≠ê1.4k)](https://github.com/Kode/Kha) - Ultra-portable, high performance, open source multimedia framework (`Web`, `Mobile`, `Desktop`, `Consoles`).
*   [Lime (‚≠ê722)](https://github.com/openfl/lime) - A flexible, lightweight layer for Haxe cross-platform developers (`Web`, `Mobile`, `Desktop`).
*   [linc\_glfw (‚≠ê16)](https://github.com/Sunjammer/linc_glfw) - Desktop - GLFW binding (multi-platform library for OpenGL, OpenGL ES and Vulkan) *(Desktop)*.
*   [NME (‚≠ê470)](https://github.com/haxenme/nme) - A cross-platform native backend (`Web`, `Mobile`, `Desktop`).
*   [3DSHaxe (‚≠ê27)](https://github.com/Krismowo/3DSHaxe) - Make 3ds homebrew! (`3DS`).

## Physics

*   [echo (‚≠ê138)](https://github.com/AustinEast/echo/) - Simple Physics Library.
*   [haxebullet (‚≠ê93)](https://github.com/armory3d/haxebullet) - Bullet 3D Physics.
*   [nape-haxe4 (‚≠ê43)](https://github.com/HaxeFlixel/nape-haxe4) - Physics Engine (the original Haxe3 version of nape can be found [here (‚≠ê542)](https://github.com/deltaluca/nape)).

## Architecture

    IoC == Inversion of Control  
    EC == Entity Component  
    ECS == Entity-Component-System
    FSM == Finite State Machine
    MVC == Model View Controller

*   [awe6 (‚≠ê81)](https://github.com/hypersurge/awe6) - `IoC`, `EC` - The inverted game framework, is a development tool focused on Future Proofing.
*   [ecx (‚≠ê51)](https://github.com/eliasku/ecx) - `ECS` - An Entity Component System framework.
*   [hexMachina (‚≠ê7)](https://github.com/DoclerLabs/hexCore) - `MVC` - A powerful multi-modular MVC framework.
*   [OSIS (‚≠ê42)](https://github.com/Dvergar/OSIS) - `ECS` - Entity Component System architecture with networking support.

## Networking

*   [Anette (‚≠ê37)](https://github.com/Dvergar/Anette) - Simple network library (no UDP).
*   [colyseus-hx (‚≠ê69)](https://github.com/colyseus/colyseus-hx) - Multiplayer Game Client.
*   [haxe-simple-peer (js) (‚≠ê4)](https://github.com/melonin/haxe-simple-peer) - Haxe externs for simple-peer.
*   [hxWebSockets (‚≠ê77)](https://github.com/ianharrigan/hxWebSockets) - Websockets for all platforms.
*   Built-in - Heaps, OpenFL (HaxeFlixel & co), Kha (Armory).

## Serialization and storage

*   [Bits (‚≠ê22)](https://github.com/RealyUniqueName/Bits) - Binary bit flags with unlimited amount of bits.
*   [CastleDB (‚≠ê517)](https://github.com/ncannasse/castle) - A structured static database easing collaboration.
*   [hxbit (‚≠ê144)](https://github.com/ncannasse/hxbit) - A binary serialization and network synchronization library.
*   [PODStream (‚≠ê21)](https://github.com/Dvergar/PODStream) - Plain Old Data serializer.

## Miscellaneous

### 3rd party API

*   [SteamWrap (‚≠ê96)](https://github.com/larsiusprime/SteamWrap) - Native extension for the SteamAPI.
*   [newgrounds](https://lib.haxe.org/p/newgrounds) - Newgrounds API.
*   [hxgamejolt-api (‚≠ê11)](https://github.com/MAJigsaw77/hxgamejolt-api) - Haxe bindings for GameJolt API.

### AI

[goap](https://gitlab.com/haath/goap) - Goal-oriented action planner for AI.

### Animation

*   [spine-hx (‚≠ê54)](https://github.com/jeremyfa/spine-hx) - Spine runtime automatically converted from the official Java/libgdx runtime.
*   HaxeFlixel - Spine parser.
*   [Heaps-Spine (‚≠ê17)](https://github.com/Beeblerox/Heaps-Spine) - Spine player for heaps.
*   [heaps-aseprite (‚≠ê25)](https://github.com/AustinEast/heaps-aseprite) - Load and render sprites and animations in Aseprite format.
*   [openfl-aseprite (‚≠ê15)](https://github.com/miriti/openfl-aseprite) - Load and render sprites and animations in Aseprite format.
*   [openfl-spine (‚≠ê16)](https://github.com/rainyt/openfl-spine) - Render Spine animation in the OpenFL engine, rendering processing can be achieved throughSprite and Tilemap.
*   [ase (‚≠ê15)](https://github.com/miriti/ase) - File format reader/writer for .ase/.aseprite without external dependencies.
*   [flxgif (‚≠ê10)](https://github.com/MAJigsaw77/flxgif) - Yagp's Gif Player for HaxeFlixel.

### Audio

*   [sfxr-hx (‚≠ê5)](https://github.com/jobf/sfxr-hx) - Pure haxe implementation of Sfxr.

### Color manipulation

*   [nxColor (‚≠ê38)](https://github.com/oscarcs/nxColor) - Color manipulation library.

### Collision

*   [differ (‚≠ê158)](https://github.com/snowkit/differ) - A separation axis theorem collision library.

### Computer Vision

*   [Vision (‚≠ê34)](https://github.com/ShaharMS/Vision) - Cross platform computer vision library.

### Data structures

*   [polygonal-ds (‚≠ê341)](https://github.com/polygonal/ds) - Data structures for games.

### Dialogues

*   [hxyarn (‚≠ê5)](https://github.com/cxsquared/hxyarn) - Parser and runner for Yarn dialogue files.

### Editor

*   [flixel-studio (‚≠ê98)](https://github.com/Dovyski/flixel-studio) - In-game editor for HaxeFlixel.

### Helpers

*   [deepnightLibs (‚≠ê215)](https://github.com/deepnight/deepnightLibs) - General gamedev purpose libs.

### Localization

*   [firetongue (‚≠ê142)](https://github.com/larsiusprime/firetongue) - A translation/localization framework.

### Map parser

*   [PyxelEdit Map Importer (‚≠ê33)](https://github.com/Dvergar/PyxelEdit-Map-Importer) - Parser for maps generated by the editor PyxelEdit.
*   Heaps - Built-in parser for Tiled.

### Math helpers

*   [hxmath (‚≠ê88)](https://github.com/tbrosman/hxmath) - A game-oriented math library.
*   [haxe-glm (‚≠ê44)](https://github.com/hamaluik/haxe-glm) - A toolset for using 2, 3, and 4 dimensional vectors and matrices, as well as quaternions.
*   [hx-vector2d (‚≠ê12)](https://github.com/markknol/hx-vector2d) - Worlds most complete Vector2d / Point class. With operator overloading.

### Modding

*   [polymod (‚≠ê147)](https://github.com/larsiusprime/polymod) - An atomic modding framework for games/apps.

### Particles

*   [Sparkler (‚≠ê15)](https://github.com/RudenkoArts/sparkler) - Modular Particle System.

### Monetization

*   [extension-iap (‚≠ê8)](https://github.com/charmdev/extension-iap) - Provides an access to in-app purchases (iOS) and in-app billing (Android) for OpenFL projects using a common API. Fork of [this (‚≠ê71)](https://github.com/HaxeExtension/extension-iap).

### Pathfinding

*   [pathfinder (‚≠ê28)](https://github.com/hypersurge/pathfinder) - Easy A\* pathfinding algorithm.
*   [astar](https://gitlab.com/haath/astar) - Versatile framework-agnostic A-star solver Library.

### Procedural generation

*   [Dungeon builder (‚≠ê64)](https://github.com/julsam/dungeon-builder) - A set of dungeon generation algorithm (works w/ hx4 w/ minor changes).

*   [TweenX/TweenXCore (‚≠ê89)](https://github.com/shohei909/tweenx) - Tween library.

### UI

*   [domkit (‚≠ê77)](https://github.com/ncannasse/domkit) - CSS Components based strictly typed UI framework.
*   [flixel-ui (‚≠ê158)](https://github.com/HaxeFlixel/flixel-ui) - GUI library for HaxeFlixel.

*   [Feathers UI](https://feathersui.com/) - Cross-platform graphical user interface components for creative frontend projects.

### Video
