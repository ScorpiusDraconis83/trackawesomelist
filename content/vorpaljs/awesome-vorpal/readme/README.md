## Extensions

*Extensions add to the functionality of Vorpal - programmatically or live.*

#### POSIX implementations

*   [less (‚≠ê13)](https://github.com/vorpaljs/vorpal-less) - Implementation of the `less` command.
*   [grep (‚≠ê12)](https://github.com/vorpaljs/vorpal-grep) - Implementation of the `grep` command.

#### Development tools

*   [repl (‚≠ê18)](https://github.com/vorpaljs/vorpal-repl) - Drops your CLI into a REPL within your app's context.
*   [tour (‚≠ê19)](https://github.com/vorpaljs/vorpal-tour) - Build an interactive tour for your Vorpal app.
*   [watch (‚≠ê3)](https://github.com/vantagejs/vantage-watch) - Updates your live Vorpal extensions in realtime.
*   [log (‚≠ê11)](https://github.com/AljoschaMeyer/vorpal-log) - Advanced logging utility.
*   [set-or-print (‚≠ê4)](https://github.com/AljoschaMeyer/vorpal-setorprint) - Easily set / get arguments with commands.

#### Realtime

*   [use (‚≠ê7)](https://github.com/vorpaljs/vorpal-use) - Import Vorpal extensions live: while your app is running.

#### Fun

*   [hacker-news (‚≠ê7)](https://github.com/vorpaljs/vorpal-hacker-news) - Pulls the top trending items from Hacker News.

#### Toolbox

*   [rc (‚≠ê7)](https://github.com/subk/vorpal-rc) - Simple .rc file support.
*   [comment (‚≠ê5)](https://github.com/subk/vorpal-comment) - Bash-like comment support.
*   [as-default (‚≠ê5)](https://github.com/ialpert/vorpal-as-default) - Triggers Vorpal task in case it runs without any specificly defined task.
